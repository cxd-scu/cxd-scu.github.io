继承

发表于 2020-11-28 更新于 2020-11-29 分类于 Java 阅读次数：

​ 继承是面向对象设计的一个基本概念，其基本思想是，可以基于已有的类创建新的类。继承原有的类就是复用这些类的方法，并增加一些新的字段与方法，使得新类能够适用新的情况。

类、超类与子类

假定一个Employee类，描述一个公司的普通员工，同时公司内存在经理这一职位，经理与普通员工有着相似的地方，如他们都会领取薪水。但普通员工在完成本职工作后仅能领取薪水，而经理在完成特定业绩后还可以领取奖金。这种情况就需要用到继承，因为需要为经理定义一个新类Manager，并增加一些新功能，但可以重用Employee类中已经编写的部分代码，并保留原Employee类中的所有字段。理论上讲，Manager与Employee之间存在着明显的“is-a”关系，每个经理都是一个员工：“is-a”关系是继承的一个明显特征。

多态

“is-a”原则指出，子类的每个对象也是超类的对象。例如，每个经理都是员工，因此，应当将Manager类设计为Employee类的子类，但反之，员工不一定是经理。

“is-a”的另一种表述是替换原则。它指出程序中出现超类对象的任何地方都可以使用子类对象替换。

在Java语言中，对象变量是多态的。一个Employee变量既可以引用一个Employee类型的对象，也可以引用Employee类任何一个子类的对象（如Manager、Secretary等）。

受保护访问

有些时候，可能限制超类中的某个方法只允许子类访问，或希望某个子类的方法访问超类的某个字段，这时需要将这些类方法或字段声明为protected。

在Java中，保护字段只能由同一个包中的类访问。

Java的四种访问控制修饰符：

  • private：仅对本类可见
  • public：对外部完全可见
  • protected：对本包与所有子类可见
  • 默认（不需修饰符）：对本包可见
Object：所有类的超类

Object类是Java所有类的始祖，每个类都扩展了Object类，定义类时如果没有明确指出超类，Object就被认为是这个类的超类。

只有基本类型不是对象，如数值、字符以及布尔值。

所有的数组类型，不管是对象数组还是基本类型的数组都扩展了Object类。

Integer、Long、Float、Double、Short、Boolean、Byte、Character

包装器类是不可变的，一旦构造了包装器，就不允许更改包装在其中的值。同时，包装器类是final的，即不允许派生它们的子类。
