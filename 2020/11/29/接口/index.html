接口

发表于 2020-11-29 更新于 2020-12-01 分类于 Java 阅读次数：

​ 接口（interface）用来描述类应该做什么，而不指定它们具体应该如何做。一个类可以实现一个或者多个接口。有些情况可能要求符合这些接口，只要有这种要求，就可以使用实现了这个接口的类的对象。

接口的概念

在Java程序设计语言中，接口不是类，而是对希望符合这个接口的类的一组需求。

例如，Arrays类中的sort方法可以对对象数组进行排序，但要求该对象所属的类必须实现Comparable接口：

这说明，任何实现Comparable接口的类必须包含一个compareTo方法，该方法有一个类型参数，并返回一个整数。准确的说，在调用x.compareTo(y)时，这个方法必须确实能够比较两个对象并返回比较的结果。当x大于y时，返回一个正数；当x等于y时，返回0；否则返回一个负数。

接口中所有的方法都自动是public的，因此，在声明时不必提供修饰符。

接口中可以包含多个方法，也可以定义常量。但是，接口不可以有实例字段，在接口中实现的简单方法也不能引用实例字段。

对象的克隆

为一个包含对象引用的变量创建副本时，原变量与副本都是同一个对象的引用，任意一个变量改变都会影响另外一个变量。 如果希望copy是一个新对象，它的初始状态与original相同，但是之后它们会有各自不同的状态，这种情况下就要使用clone方法。

不过，clone方法是Object的一个protected方法，这说明你的代码不能直接调用这个方法，因为Object对你编写的类一无所知，只能逐个字段的进行拷贝。如果对象中所有数据字段都是基本类型，拷贝就没有任何问题。但如果对象包含子对象的引用，拷贝字段就会得到相同子对象的另一个引用，这样，原对象与克隆对象仍会共享一些信息。
